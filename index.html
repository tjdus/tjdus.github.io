프로젝트

학회 관리 플랫폼 CNU&U

서비스 바로가기
2026.1–2026.2 · 1인 개발 · 백엔드/프론트엔드 · Spring Boot · Next.js · PostgreSQL

프로젝트 개요

노션, 구글폼, 디스코드 등에 분산되어 있던 학회 운영 데이터를 통합하고,
모집·신청·활동 관리를 하나의 서비스로 통합한 학회 관리 웹 서비스입니다.

주요 기능

  • 학회원 관리 및 학기별 활동 관리
  • 모집 공고 및 신청 관리
  • 신청 응답 통계 및 관리 기능
  • 활동 회차별 출석 관리 및 수료 여부 확인

기여 내용

  • 학회원·활동·신청·모집 데이터를 관리하는 DB 스키마 설계
  • 모집, 신청, 활동 관리 기능을 포함한 전체 서비스 개발
  • 회차별 출석 관리 및 활동 수료 여부 확인 기능 구현
  • 신청 응답 통계 및 관리 기능 구현
  • 실제 학회 운영에 서비스 적용 및 운영
문제 해결

학회 운영 데이터 통합

문제
  • 노션, 구글 폼 등 여러 플랫폼에 분산된 데이터로 인한 운영 비효율
해결 방법
  • 회원·모집·신청·활동 데이터를 유기적으로 연결하는 통합 DB 스키마 설계
  • 모집 공고부터 수료 처리까지의 전체 워크플로우를 웹 서비스로 구현
결과
  • 수작업으로 진행되던 반복 업무를 제거해 데이터 정합성 확보
  • 실제 학회 운영에 도입되어 매 학기 100명 이상의 회원 관리에 활용 중

활동 관리 및 출석 확인 화면

서비스 운영을 위한 배포 자동화

해결 방법
  • Docker를 활용한 일관된 실행 환경 구축 및 GitHub Actions 기반 자동화 파이프라인 설계

하루 5분 주식 리포트 UPssue

GitHub 저장소
2025.10–2025.12 · 산학 프로젝트 · 5명 · 백엔드/프론트엔드 · FastAPI · Kafka · RabbitMQ

프로젝트 개요

UPssue는 주식 시장에서 급등·급락한 종목의 변동 원인을
뉴스와 데이터를 기반으로 분석해 근거 중심 AI 리포트를 자동 생성하는 서비스입니다.

주요 기능

  • 급등·급락 종목의 상승·하락 원인을 뉴스 근거 기반으로 자동 분석
  • 테마 내 관련 종목의 상승 가능성 예측 및 종목 간 관계를 고려한 파급력 분석
  • 매일 주요 경제 뉴스와 핵심 이슈 요약 리포트 제공

기여 내용

  • 대용량 뉴스 수집·가공·분석·리포트 생성 흐름을 파이프라인으로 설계 및 구현
  • 사업보고서 데이터를 활용해 종목 간 관계 그래프를 구성하고 관계 정보를 반영
  • 일별 증시 요약 리포트 생성 흐름 설계 및 자동화 구현
문제 해결

데이터 트래픽 증가로 인한 지연과 안정성 한계

문제
  • 대량 뉴스 데이터 처리 시 순차적 파이프라인 구조로 인한 심각한 병목 현상 발생
  • 파이프라인 중간 단계 오류 시 실패 지점 추적 및 부분 재처리가 불가능해 비효율 촉발
해결 방법
  • 메시지 큐(Kafka, RabbitMQ)를 도입해 크롤링, 분석, 임베딩 등 각 작업 단계를 독립적으로 분리
  • 단계별 메시지 큐와 토픽을 정의하여 안정적인 비동기 스트리밍 아키텍처 구축
결과
  • 시스템 부하가 심한 분석 작업만 개별적으로 서버를 늘려 유연하게 대응할 수 있게 됨
  • 전체 과정을 다시 실행할 필요 없이, 오류가 발생한 단계의 데이터만 재처리 가능해짐

메시지 큐 기반 비동기 처리 아키텍처

파급력 예측 맥락 부족

문제
  • 거래량 등 단순 정량 지표만으로는 특정 종목의 이슈가 연관 종목으로 전파되는 과정을 파악하기 어려움
해결 방법
  • 사업보고서를 바탕으로 기업 간 관계(계열사, 공급망 등)를 연결한 GNN(그래프 신경망) 모델 구축
  • 특정 종목이 오를 때 연관된 다른 종목도 오를 가능성을 관계 기반으로 예측하도록 구현
결과
  • 특정 종목의 변동이 연관 종목으로 파급되는 원인과 실질적 관계를 근거로 제시하여, 예측 결과의 논리적 설명력 향상

종목 관계 그래프 예시

ESG 배출 데이터 대시보드

GitHub 저장소
2025.01 · 부트캠프 · 4명 · 백엔드/프론트엔드 · Next.js · Express · Handsontable · Chart.js

프로젝트 개요

ESG 배출량 데이터를 기업·사업장 단위로 관리하고,
Scope(1/2/3) 기준 배출량 계산과 데이터 수정 이력을 추적할 수 있는 ESG 대시보드 서비스입니다.

주요 기능

  • 기업·사업장 단위의 배출량 데이터 관리 및 조회
  • Scope(1/2/3) 기준 배출량 계산 및 시각화
  • 대량 배출 데이터 일괄 입력 및 편집 지원

기여 내용

  • 기업·사업장·배출 데이터 및 수정 이력에 대한 DB 스키마 설계
  • Scope 기반 배출량 계산 로직 및 관련 API 구현
  • 대량 배출 데이터 입력·수정 처리 API 개발
  • Handsontable 기반 데이터 입력 화면 구현 및 대시보드 시각화
문제 해결

대량 배출 데이터 처리 구조 설계

문제
  • 대규모 배출량 데이터의 수정 내역을 추적하기 어려워 감사(Audit) 대응에 한계가 있음
  • 데이터 변경 시 연관된 Scope(1/2/3) 계산 결과와의 정합성 보장 필요
해결 방법
  • 원본 입력 데이터, 계산 결과, 수정 이력을 완전히 분리하여 관리하는 아키텍처 도입
  • 수정 시점에 스냅샷을 남기는 이력 관리 API 및 비동기 재산정 로직 구축
결과
  • 모든 변경 사항 추적 및 복원이 가능해져 엔터프라이즈급 데이터 신뢰성 확보
  • 엑셀 기반 대량 데이터 일괄 입력 환경에서도 흔들림 없는 계산 정합성 유지
