Challenge 1: 학회 운영 데이터 통합

Problem
  • 모집, 신청, 활동 기록이 여러 시스템에 분산되어 데이터 일관성 유지와 효율적 관리 어려움
Approach
  • 학회원, 모집, 신청, 활동 데이터를 하나의 DB 구조로 통합 설계
  • 모집 → 신청 → 활동 관리로 이어지는 운영 흐름을 서비스 기능으로 구현
  • 설문 기반 신청 시스템(단답형, 장문형, 단일 선택, 복수 선택)을 서비스 내에서 구현
  • 학회원별 분기 활동 참여 이력을 관리하고 자동 수료 계산 기능 구현
  • 활동 페이지에서 출석 관리와 수료 여부를 함께 확인할 수 있도록 설계
  • 온라인 강의 예약, 보고서 제출, 관리자 시간표 관리 기능을 하나의 서비스로 통합
Result
  • 학회 운영 기능을 하나의 서비스로 통합하여 운영 효율성 개선
  • 학회원 활동 데이터를 체계적으로 관리할 수 있는 구조 구축
  • 실제 학회 운영에 적용하여 지속적으로 사용되는 서비스로 운영

Challenge 2: Weak Impact Prediction Context

Problem

Result
